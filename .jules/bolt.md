## 2024-03-12 - Chart Component Array Re-Allocations
**Learning:** React components dealing with large portfolio sets like `AllocationChart` and `SectorDistribution` perform array sorting `O(N log N)` and mapping operations directly in the render body. Because these operations create new object and array references on *every render*, they likely pass new references down to heavy charting components (like Recharts' `PieChart`), destroying their ability to bail out of re-renders.
**Action:** When working with data-heavy components passing props to complex visualizers (like charts or tables), aggressively wrap data transformations in `useMemo` keyed off the raw data (e.g., `holdings`). This prevents reference churn and stabilizes performance during extraneous component re-renders.
