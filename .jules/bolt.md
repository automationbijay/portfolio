## 2024-05-18 - [Memoizing Dashboard Component Computations]
**Learning:** Found an opportunity to improve frontend performance by memoizing computations on portfolio data using React's `useMemo`. In components like `AllocationChart`, `SectorDistribution`, and `TradingHistoryCard`, arrays such as `holdings` were being sorted, sliced, mapped, and reduced directly within the render loop. This meant on every state change or re-render (e.g. daily price updates), these expensive O(N log N) / O(N) operations were unnecessarily re-running.
**Action:** Used `useMemo` hooks with strict dependency arrays (`[holdings]`, `[tradingHistory, dividendDetails]`) to wrap these operations. This pattern ensures derived data structures are only re-calculated when the underlying data actually changes, conserving main thread resources. Added bolt optimization comments to keep the code's intent clear.
